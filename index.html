3D Web Programming Toolkit
for BIM and Engineering Visualization

xeokit is a software development kit from xeolabs for interacting with high-detail 3D engineering models in the browser.

The open source xeokit SDK provides a high-performance, purpose-built WebGL-based 3D engine with a bundled library of plugins focused on BIM and engineering. The engine uses advanced rendering techniques to view and interact with real-world engineering models in most major browsers, on both desktop and mobile platforms.

xeokit Features

Display your Models in the Browser

Rapidly build your high-performance engineering visualization application.

  • Load large models and metadata quickly.
  • Pick, move, highlight, x-ray, slice, measure and annotate objects.
  • Load models from IFC, glTF, OBJ, STL, 3DXML, XKT and BIMServer.
  • Save and load sharable BCF viewpoints.
  • Works in all major browsers, including mobile, without plugins.
  • Zero external dependencies.

JavaScript Programming Toolkit

An open source toolkit of solutions based on 10+ years experience visualizing 3D BIM and STEM on the Web.

  • Purpose-built WebGL graphics engine.
  • Interactively renders high-detail models.
  • Scene graph, transforms, animations, physically-based materials.
  • Clean, modular JavaScript ES6.
  • Complete documentation and tutorials.

Plugins and Tools Library

A growing library of plugins and pipeline tools to accelerate the development of your application, including:

  • Sectioning, annotation, distance measurement, angle measurement, BCF collaboration
  • Plan views, tree views, NavCube, gnomon
  • glTF loader, OBJ loader, STL loader, XKT loader, 3DXML loader, BIMServer loader

Expert Community

Join an open community of 3D Web programming experts. Share our tools and get support.

What xeokit's users say

xeokit is a success enabler for our customers. Incredibly fast load times and super high-performance rendering, even on mobile browsers. Built in a modular way that allowed us to easily customize its behaviour, while allowing us to create a differentiated navigation experience.

Clean code, with the ability to inspect its internals, was key to our development experience. Teams focused on simple things can start using it with less than ten lines of JavaScript, while more advanced users will find that xeokit can be adapted to meet their advanced use cases.
