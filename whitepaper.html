Run this example application [here](https://xeokit.github.io/xeokit-viewer/).* # Introduction [xeokit](http://xeokit.io) is an open source software development kit from [xeolabs](http://xeolabs.com) for viewing high-detail 3D models in the browser. The SDK includes a purpose-built WebGL-based[^webgl] 3D engine with a bundled library of plugins focused on BIM [^bim] and AEC [^aec] visualization. The engine loads models quickly and uses advanced rendering techniques to interactively view large numbers of objects in most major browsers, on both desktop and mobile platforms. The SDK is provided under an Affero GPL V3 dual-license, which allows free use for non-commercial purposes, with the option to purchase a licence for commercial use. See [Pricing](https://xeokit.github.io/xeokit-licensing/) for commercial options. [^webgl]: WebGL is a JavaScript API for rendering interactive 2D and 3D graphics within any compatible web browser without the use of plug-ins [^bim]: Building Information Modeling (BIM) [^aec]: Architecture, Engineering and Construction (AEC) Features Overview =============================================================================== ## Core SDK Features * **Load large models quickly** from one of the fastest-loading geometry formats on the Web. * **Render high-detail interactively** in all major Web browsers, using one of the highest-performing purpose-built WebGL-based 3D renderers on the market. * **Load IFC metadata** (IFC2x3 and IFC4) with your models to classify objects and assist navigation. Load models from your own data store, or straight off the file system. No need to store models in the cloud. * **Completely open source** - customize the code and integrate with your own data pipeline - don't get locked in to high IFC-to-Web conversion costs with closed systems. * **Rapidly build applications** using a growing library of plugins that fast-track the development of your application. * **Physically-based rendering** for realistic previews. * **Dynamic scene graph** representation for animations. * **Complete API documentation** covers everything in the SDK. * **Live examples** demonstrate all features and provide boilerplate code to get you started. * **Clean, readable code** following good software design principles, in modern JavaScript (ECMA6). * **Used and supported by industry leaders** who sustain development through licensing, Technical Details =============================================================================== ## Fast Loading **The xeokit SDK's fastest loading format is its native binary .XKT format**, which is a single BLOB containing a model, compressed using geometry quantization and DEFLATE compression. Open source CLI tools are available to convert IFC STEP, glTF and DAE files into .XKT format for efficient loading by xeokit. CLI tools are also available to extract JSON IFC metadata, to load alongside the model geometry. These tools can be easily scripted to build automated IFC -> xeokit data pipelines. xeokit can also import models from glTF, OBJ, STL, 3DXML and BIMServer, and can load multiple models into the same scene from different sources. This includes xeokit's viewer, plugins, and pipeline tools. Being open source reduces risk for users, and affords them the ability to inspect the code, fork the repository and make any necessary fixes and customizations. xeokit's development process is fully transparent. Users can follow and participate in development on GitHub[^github], and receive news of new developments on Twitter[^twitter]. [^GitHub]: [https://github.com/xeokit/xeokit-sdk](https://github.com/xeokit/xeokit-sdk) [^twitter]: [#xeogl on Twitter](https://twitter.com/hashtag/xeokit?src=hashtag_click&f=live) ## Open Data Pipeline xeokit does not box users into a closed pipeline. The SDK allows to load models from the filesystem [^filepipe], or, using custom loading strategies, from your own data source, including BIMServer [^bimserver]. Many other solutions keep user's models on their own servers, while incurring a high IFC-to-Web conversion cost per model. Some users have developed their own strategies for efficiently loading models into xeokit, such as progressively streaming models from their own model server.
