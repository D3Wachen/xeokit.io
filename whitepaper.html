Run this example application [here](https://xeokit.github.io/xeokit-viewer/).* # Abstract [xeokit](http://xeokit.io) is an open source software development kit from [xeolabs](http://xeolabs.com) for viewing high-detail 3D models in the browser. The SDK includes a purpose-built WebGL-based[^webgl] 3D engine with a bundled library of plugins focused on BIM[^bim] and AEC[^aec] visualization. The engine loads models quickly and uses advanced rendering techniques to interactively view large numbers of objects in most major browsers, on both desktop and mobile platforms. The SDK is provided under an Affero GPL V3 dual-license, which allows free use for non-commercial purposes, with the option to purchase a licence for commercial use. [^webgl]: WebGL is a JavaScript API for rendering interactive 2D and 3D graphics within any compatible web browser without the use of plug-ins [^bim]: Building Information Modeling [^aec]: Architecture, engineering and construction Features Overview =============================================================================== ## Core SDK Features * **Load large models quickly** from one of the fastest-loading 3D geometry formats on the Web. * **Render high-detail interactively** in all major Web browsers, using one of the most performant purpose-built WebGL-based 3D renderers on the market. * **Load IFC metadata** alongside your models to classify objects and assist navigation. Load BIM models straight off the file system, no need to store models in the cloud. * **Rapidly build applications** using a growing library of plugins that fast-track application development for BIM and engineering. * **Physically-based rendering** capability for realistic previews. * **Scene graph** represntation * **Complete API documentation** covers everything in the SDK. * **Live examples** demonstrate all features and provide boilerplates to get you started. * **Clean readable code** developed with empathy for developers, following good software design principles, in modern JavaScript (ECMA6). * **Used and supported by industry leaders** who sustain development through licensing, sponsorship and code contributions. ## Plugins * **Storey plan views** * **Sectioning** * **BCF viewpoints** * **Camera navigation gizmos** Technical Details =============================================================================== Fast Loading ------------------------------------------------------------------------------- **The SDK's fastest loading format is its native binary .XKT format**, which is a single BLOB containing a model, compressed using geometry quantization and DEFLATE compression. Open source CLI tools are available to convert IFC STEP and glTF files into .XKT format for efficient consumption by xeokit. CLI tools are also provided to extract JSON IFC metadata to load alongside the model geometry. These tools can be easily scripted to build automated IFC -> xeokit data pipelines. xeokit can also import models from glTF, OBJ, STL, 3DXML and BIMServer, and can load multiple models into the same scene from different sources. This includes xeokit's viewer, plugins, and pipeline tools. This reduces risk for users, and affords them the ability to inspect the code, fork the repository and make any necessary fixes and customizations. xeokit's development process is fully transparent. Users can follow and participate in development on GitHub[^github], and receive news of new developments on Twitter[^twitter]. [^twitter] [#xeogl on Twitter](https://twitter.com/hashtag/xeokit?src=hashtag_click&f=live)
[^GitHub] [https://github.com/xeokit/xeokit-sdk](https://github.com/xeokit/xeokit-sdk) ## Open Data Pipeline xeokit does not box users into a closed pipeline. The SDK allows to load models from the filesystem[^filepipe], or, using custom loading strategies, from your own data source, including BIMServer. AutoDesk Forge, by comparison, keeps user's models on their own servers, while incurring a high IFC-to-Web conversion cost per model. Some users have developed their own loading strategies, including one that progressively streams models into xeokit from their own model server. [^filepipe][Creating Files for Serverless BIM](https://github.com/xeokit/xeokit-sdk/wiki/Creating-Files-for-Offline-BIM) [^bimserver][BIMServerLoaderPlugin](https://xeokit.github.io/xeokit-sdk/docs/class/src/plugins/BIMServerLoaderPlugin/BIMServerLoaderPlugin.js~BIMServerLoaderPlugin.html) ## Unmatched Performance The xeokit SDK currently fills a gap in the market for performant viewing of large, real-world BIM models in the browser. Most other toolkits, such as Autodesk Forge, are built around general-purpose WebGL libraries, such as THREE.js, which are designed for maximum extensibility in order to support a wide range of visual effects capabilities, while xeokit is specifically designed from the ground up to render large models for BIM and engineering [^instancing], [^batching]. When you use xeokit, you'll be tapping into xeolabs ongoing research into high-performance Web visualization (for example, xeokit plans to support WebGPU).
